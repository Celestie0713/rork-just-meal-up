# Google Places Integration\n\nThis app includes a complete Google Places integration for finding and displaying restaurants and other places.\n\n## Features\n\n- **Search Places**: Search for restaurants and places by text query\n- **Nearby Places**: Find places near your current location\n- **Place Details**: View detailed information including photos, reviews, ratings, and contact info\n- **Location Services**: Automatic location detection with permission handling\n- **Mock Data**: Works with mock data when Google Places API is not configured\n\n## Setup Instructions\n\n### 1. Get Google Places API Key\n\n1. Go to the [Google Cloud Console](https://console.cloud.google.com/)\n2. Create a new project or select an existing one\n3. Enable the following APIs:\n   - Places API\n   - Places API (New)\n4. Create credentials (API Key)\n5. Restrict the API key to the Places API for security\n\n### 2. Configure API Key\n\nOpen `services/google-places.ts` and replace:\n```typescript\nconst GOOGLE_PLACES_API_KEY = 'YOUR_GOOGLE_PLACES_API_KEY';\n```\n\nWith your actual API key:\n```typescript\nconst GOOGLE_PLACES_API_KEY = 'your_actual_api_key_here';\n```\n\n### 3. Test the Integration\n\n1. Run the app\n2. Go to the Search tab\n3. Switch to the \"Places\" tab\n4. Grant location permission when prompted\n5. Search for restaurants or browse nearby places\n\n## File Structure\n\n```\ntypes/\n  place.ts              # TypeScript interfaces for places\nservices/\n  google-places.ts      # Google Places API service\nhooks/\n  use-places.ts         # React hook for places functionality\ncomponents/\n  PlaceCard.tsx         # Place card component\nmocks/\n  places.ts             # Mock data for development\napp/\n  place-details.tsx     # Place details screen\n  (tabs)/\n    index.tsx           # Updated search screen with places\n```\n\n## Usage\n\n### Search Places\n- Switch to the \"Places\" tab in the search screen\n- Type in the search box to find specific restaurants or cuisines\n- Leave search empty to see nearby places\n\n### View Place Details\n- Tap on any place card to view detailed information\n- See photos, ratings, reviews, contact info, and opening hours\n- Use action buttons to get directions, call, or visit website\n\n### Location Services\n- The app will request location permission on first use\n- Location is used to find nearby places and improve search results\n- Works without location but with limited functionality\n\n## Mock Data Mode\n\nWhen the API key is not configured, the app automatically uses mock data:\n- Shows sample restaurants with realistic data\n- Includes photos from Unsplash\n- Simulates API delays for realistic testing\n- Perfect for development and testing\n\n## API Costs\n\nGoogle Places API has usage-based pricing:\n- Text Search: $32 per 1000 requests\n- Nearby Search: $32 per 1000 requests\n- Place Details: $17 per 1000 requests\n- Photos: $7 per 1000 requests\n\nConsider implementing caching and request optimization for production use.\n\n## Customization\n\n### Adding More Place Types\nModify the search parameters in `hooks/use-places.ts`:\n```typescript\nsearchNearbyPlaces(lat, lng, 1500, 'restaurant|cafe|bar');\n```\n\n### Customizing Place Cards\nEdit `components/PlaceCard.tsx` to change the appearance and information displayed.\n\n### Adding Filters\nImplement filtering by cuisine, price range, rating, etc. in the search screen.\n\n## Troubleshooting\n\n### \"Location Access Needed\" Screen\n- Make sure location permissions are granted in device settings\n- Check that location services are enabled\n- Try restarting the app after granting permissions\n\n### No Places Found\n- Verify your API key is correct and has Places API enabled\n- Check the browser/device console for error messages\n- Ensure you have internet connectivity\n- Try searching in a different location\n\n### API Errors\n- Check your Google Cloud Console for API quotas and billing\n- Verify the API key restrictions allow your domain/app\n- Monitor the console logs for specific error messages
